# Changelog

All notable changes to sourcebook are documented here.

## [0.9.0] — 2026-04-08

### Added
- **AGENTS.md generated by default** — `sourcebook init` and `sourcebook update` now output both `CLAUDE.md` and `AGENTS.md` without any flags. Works with Claude Code, Cursor, GitHub Copilot, OpenAI Codex, and Windsurf.
- **Python import graph analysis** — sourcebook now builds a real import graph for Python repos, identifying hub files by fan-in count (threshold: 3 imports for Python vs. 5 for TS/JS). Pydantic, FastAPI, Django repos now get accurate hub detection.
- **Library mode detection** — repos with `pyproject.toml` + flat package layout or `src/` layout are automatically classified as libraries. The "publishable library, not an application" constraint is injected into output — agents no longer need to be told this manually.
- **`sourcebook update` agents support** — `update` command now handles the `agents` format with full merge/preserve logic, same as `claude` and `copilot`.
- **FastAPI security auth patterns** — `OAuth2PasswordBearer`, `HTTPBearer`, `APIKeyHeader` now recognized as auth patterns.
- **Python entry points in Tier 1 sampling** — `__init__.py` and `main.py` guaranteed in the file sample for Python repos.

### Fixed
- **Docs/examples exclusion** — `docs/`, `docs_src/`, `examples/`, `benchmarks/` directories excluded from both source file sampling and hub detection. Eliminates false positives on repos with large documentation trees.
- **i18n false positive** — `_("string")` translation calls no longer flagged as auth patterns. Lookbehind guard added: only matches `_(` not preceded by a word character.
- **Confidence floor** — findings with 3+ evidence files are promoted from `low` to `medium` confidence. Reduces noise in output for well-evidenced findings.
- **Cycle display** — `__init__.py` files now shown as `dir/__init__.py` in circular dependency output instead of bare `__init__.py`.

### Changed
- Default output format changed from `claude` to `claude,agents` across all commands (`init`, `update`, `diff`, `watch`).

## [0.8.3] — 2026-04-07

- MCP server mode (`sourcebook serve`) — exposes codebase context to AI tools via the Model Context Protocol
- `sourcebook watch` — auto-regenerates context files on source changes
- `sourcebook diff` — shows what would change if context files were regenerated
- Pro license system with `sourcebook activate`
