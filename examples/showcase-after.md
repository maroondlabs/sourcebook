## Conventions & Patterns

- **Project structure:** This is a monorepo (Turborepo). Changes may affect multiple packages. Check workspace dependencies before modifying shared code. packages/ui is consumed by both apps/web and apps/dashboard — test both after UI changes.
- **Import conventions:** Project uses barrel exports (47 index.ts files that re-export). Import from the package name, not from deep paths. Use `@acme/ui` not `@acme/ui/src/primitives/Button`.
- **Import conventions:** Strongly prefers path alias imports (@/ over relative). 312 alias imports vs 41 relative imports in sampled files. Use `@/components/X` not `../../components/X`.
- **Export conventions:** Project strongly prefers named exports over default exports. 489 named vs 12 default exports. Use `export function` / `export const`, not `export default`. The only exceptions are Next.js page components (required by framework).
- **Next.js routing:** Project uses App Router exclusively. All routes are in apps/web/src/app/. Server components by default. Add `"use client"` only when you need interactivity — don't add it preemptively.
- **Next.js images:** Remote image patterns are configured in next.config. New image domains must be added to remotePatterns before use. Agents will get 400 errors otherwise.
- **Supabase:** Local Supabase setup detected (supabase/ directory). Use `supabase db push` for migrations during development, not the dashboard. RLS policies are enforced — every new table needs explicit policies.
- **Commit conventions:** Uses Conventional Commits (feat/fix/docs). Common scopes: web, db, auth, ui, trpc, email. Follow this pattern for new commits.
- **TypeScript imports:** Path aliases configured: @acme/ui, @acme/db, @acme/auth, @acme/email, @acme/config, @/. Use these instead of relative imports.
- **TypeScript:** Strict mode is ON across all packages. Don't weaken type assertions with `as any` or `@ts-ignore` — fix the types instead.
