FAQ

What's the difference between init and update?

init generates fresh context files (free). update re-analyzes your codebase while preserving any sections you added manually (Pro). If you've added architectural decisions, team conventions, or domain knowledge to your CLAUDE.md, update keeps those and refreshes everything else.

What's MCP server mode?

sourcebook serve runs a local MCP server that AI agents can query on-demand. Instead of reading a static file, agents ask questions like "what's the naming convention for hooks?" and get targeted answers. Works with Claude Code, Cursor, and any MCP-compatible tool.

Does any data leave my machine?

No. All analysis runs locally. The only network call is license key validation (once per 24 hours, sends just the key — no code, no file contents, no repo data). The web demo feature runs server-side but processes public GitHub repos only.

Can I cancel anytime?

Yes. Cancel from Stripe's customer portal. Your Pro features stay active until the end of your billing period. Context files you've already generated are yours to keep — they're local files on your machine.

Why BSL instead of MIT?

The core analysis is free to use, modify, and self-host. BSL prevents competitors from reselling sourcebook as a hosted service. The license converts to MIT in 2030. This is the same model used by Sentry, MariaDB, and CockroachDB.
