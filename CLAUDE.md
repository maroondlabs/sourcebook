## Architecture

The scanner runs in two passes: structural (import graph via PageRank) first, then git history. PageRank results inform pattern sampling ‚Äî don't merge the passes. The tool is stateless: no caching layer, no framework, no ORM. Side effects are filesystem reads and git shell calls only.

`src/generators/claude.ts` takes a `ProjectScan` and renders it into a CLAUDE.md string. It enforces a token budget and applies context-rot-aware section ordering ‚Äî critical findings go at the top and bottom, never the middle.

The default output is non-verbose: discoverable context (standard test frameworks, obvious stack) is stripped. `--verbose` restores it. Don't change this default ‚Äî it's the core research finding.

`sourcebook check` (Layer A) runs static analysis first. `--ai` adds Layer B (LLM completeness check) only if requested. Check-first is the product positioning ‚Äî don't conflate the two layers.
